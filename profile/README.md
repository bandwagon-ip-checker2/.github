## 为什么搬瓦工 IP 会被封？

说之前先理解一下背景。

搬瓦工的 IP 有两种"被封"的情况，性质不一样：

**一种是被 GFW 墙了。** 这是最常见的情况，属于国内网络防火墙主动屏蔽了这个 IP。从国内访问不了，但从国外节点测试是完全正常的。原因通常是该 IP 被用于某些敏感用途，触发了 GFW 的过滤机制。

**另一种是被搬瓦工官方封禁。** 这相对少见，通常是因为滥用资源（比如长期跑高 CPU、发送垃圾邮件）或违反搬瓦工服务条款导致的。如果是这种情况，官方 KiwiVM 面板会直接告诉你。

两种情况的检测方式不同，解决方法也不同，下面分开说。

## 搬瓦工检测 IP 的三种方法

### 方法一：搬瓦工官方 KiwiVM 黑名单检测工具

这是最权威的方法，直接查看你的 IP 是否被搬瓦工官方列入黑名单。

操作步骤如下：

1. 登录搬瓦工账户，进入「Services → My Services」，找到你要检测的 VPS，点击进入 KiwiVM 控制面板。
2. 进入 KiwiVM 后，注意浏览器地址栏，把地址末尾的 `main.php` 替换为 `main-exec.php?mode=blacklistcheck`，回车加载。
3. 页面中间会有一个蓝色「Test Main IP」按钮，点击即可开始检测。

结果只有两种：

- **IP NOT BLOCKED**：IP 正常，官方层面没有问题。
- **IP BLOCKED**：IP 已被官方封禁，通常是违规使用导致的。

需要注意的是，这个工具只检测"官方是否封禁"，不能直接判断 IP 是否被 GFW 墙了。如果结果显示 NOT BLOCKED 但你仍然连不上，就需要用下面的方法继续排查。

### 方法二：port.ping.pe——境内外连通性对比测试

这是判断 IP 是否被 GFW 屏蔽最直观的方式。

打开 [port.ping.pe](http://port.ping.pe)，输入你的 VPS IP 和 SSH 端口（格式为 `IP:端口`），点击测试。

页面会分别从全球多个节点（包括国内和国外）发起连接请求，结果分为 successful（成功）和 failed（失败）两种。

判断逻辑很简单：

- **国外节点全绿（successful），国内节点全红（failed）**：IP 被 GFW 墙了，基本确认。
- **国内外都绿**：完全正常，问题出在别处，检查一下本地网络或者服务器配置。
- **国内外都红**：服务器本身有问题，可能没开机或者端口没放行，检查 VPS 状态。

有时候测试结果里会有「pinger offline」的字样，这是测试节点自身的问题，和你的 VPS 无关，直接忽略就好。

### 方法三：国内 + 国外双重端口扫描法

这是最传统的方法，适合当上面两种方式都无法判断时用作最终确认。

**第一步，国内扫描：** 打开 `tool.chinaz.com/port`，输入 VPS IP 和 SSH 端口，点击开始扫描。如果结果显示"开放"，那 IP 没有问题；如果显示"关闭"，继续下一步。

**第二步，国外扫描：** 打开 `yougetsignal.com/tools/open-ports/`，输入同样的 IP 和端口，点击 Check。

结果分析：

- 国内关闭 + 国外开放 → **IP 被封了**，需要换 IP。
- 国内关闭 + 国外也关闭 → **服务器本身的问题**，检查 VPS 是否正常运行。
- 国内开放 → IP 完全正常，排查其他原因。

**Q：只靠 ping 能判断 IP 是否被封吗？**

不能，而且这个方法已经过时了。现在的情况是很多被封 IP 依然可以 ping 通（因为 GFW 只封了 TCP 端口，ICMP ping 没有被拦截），所以 ping 通并不代表 IP 正常可用。要用端口扫描的方式来测，也就是上面说的方法二和方法三。
